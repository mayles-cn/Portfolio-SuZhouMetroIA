# 苏州地铁智能助手 - 项目简历描述

## 项目名称

苏州地铁智能助手（SuZhouMetroIA）

## 一句话介绍

一个基于 Qt Widgets 与本地地铁数据的智能出行与购票演示系统，集成地图交互、语音问答、模型函数调用与结算流程。

## 项目简介

该项目面向“地铁咨询 + 购票引导”场景，提供从自然语言提问到路线与票价反馈，再到结算出票的完整交互闭环。系统支持文本与语音输入，内置地铁领域工具函数（线路查询、站点线路归属、票价估算），并通过可视化地图与路线图增强信息可读性。项目还实现了 JSON 化样式中心，降低 UI 维护成本，提高迭代效率。

## 技术栈

- C++17
- Qt6（Widgets / Multimedia / Network / 可选 WebSockets）
- CMake
- JSON 配置驱动（模型规则、公告、样式）

## 核心模块

1. 地图模块：线路绘制、站点选中、视图缩放与焦点过渡。
2. 聊天模块：文本/语音输入、流式回复、状态气泡管理。
3. 工具模块：地铁线路清单、站点归属、票价估算。
4. 结算模块：预填订单、数量调节、模拟支付、出票信息。
5. 配置模块：模型配置、助手规则、样式配置统一管理。

## 个人贡献亮点（可用于简历）

1. 设计并实现模型工具调用闭环：模型请求 -> 工具执行 -> 结果回灌 -> 最终回复。
2. 优化语音输入体验：分段转写合并为单条消息，避免重复提交。
3. 落地 JSON 样式中心（`ui_styles.json` + `UiStyleService`），将分散样式统一收敛。
4. 完成地图、快速购票、结算联动流程，形成可演示的完整业务路径。
5. 建立运行时文档、函数调用文档与质量门禁文档，提升工程可维护性。

## 项目结果

1. 形成可运行的 Windows 桌面端演示程序。
2. 支持多入口购票（地图“去这里”与快速购票栏）。
3. 实现配置化样式管理，后续 UI 调整无需频繁改动业务代码。
4. 工程结构清晰，适合作为智能交通/智能客服类项目样例。

## 可量化描述（简历可选）

1. 统一接管多个核心控件样式到 JSON 配置中心，降低样式改动的跨文件维护成本。
2. 打通文本/语音/地图/结算四条链路，构建端到端交互闭环。
3. 建立函数调用文档与质量门禁清单，提升协作开发与回归测试效率。
